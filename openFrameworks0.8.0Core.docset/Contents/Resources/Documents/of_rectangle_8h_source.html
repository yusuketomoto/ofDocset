20 // ofRectangle is a simple container for describing the position
21 // and size of 2D rectangles. Like many rectangle data structures
22 // found in other frameworks and graphics libraries, member
23 // width and height variables can take negative values.
24 // Additionally, x / y position and width / height are publicly
25 // accessible, resulting in great programming freedom.
26 // Consequently, two visually identical rectangles can be
27 // represented in the following ways:
28 //
29 // ofRectangle myRect(0,0,100,100);
30 //
31 // AND
32 //
33 // ofRectangle myRect(100,100,-100,-100);
34 //
35 // While both representations will yield the same visual
36 // results in all openFrameworks renderers, the results of
37 // some method operations that modify x / y / width / height
38 // (such as scaling) produce mathematically correct, but
39 // visually different results for each of the above
40 // representations.
41 //
42 // That said, most `getter` methods return the same results
43 // for both "standardized" and "non-standardized" rectangles.
44 // Simply stated, the user must be aware of the differences
45 // between these two visually equal but mathematically different
46 // representations.
47 //
48 // To ensure consistent results, users are encouraged to make
49 // sure rectangles are "standardized" before using operations
50 // that modify x / y / width / height.
51 //
52 // "Standardized" rectangles are rectangles whose width >= 0 and
53 // height >= 0. The `void standardize()` method can be used
54 // to ensure that the rectangle is "standardized".
