15 Description
16 -----------
17 
18 ofxKinect is an Open Frameworks addon for the Xbox Kinect that runs on Mac OSX, Linux, and Windows.
19 OpenFrameworks is a cross platform open source toolkit for creative coding in C++.
20 
21 [http://www.openframeworks.cc/](http://www.openframeworks.cc/)
22 
23 #### NOTE: Xbox Kinect models > 1414 (1473, etc) or Kinect4Windows devices that have been plugged into an XBox will not work with ofxKinect on Mac OSX
24 
25 If you have an Xbox Kinect model 1473+ or Kinect4Windows device and want to use it with ofxKinect on OSX, DO NOT plug it into an XBox! If you do, it will receive a firmware update which will cause it to freeze after about 20 secs with libfreenect/ofxKinect on OSX. Linux and Windows are fine. The [fix is in the works](https://github.com/OpenKinect/libfreenect/pull/325).
26 
27 In the meantime, we suggest you get the *original* Xbox Kinect model 1414 and keep those new toys away from an XBox ...

29 Installation
30 ------------
31 
32 To use ofxKinect, first you need to download and install [Open Frameworks](https://github.com/openframeworks/openFrameworks).
33 
34 To get a copy of the repository you can download the source from [http://github.com/ofTheo/ofxKinect](http://github.com/ofTheo/ofxKinect/) or, alternatively, you can use git clone:
35 <pre>
36 git clone git://github.com/ofTheo/ofxKinect.git
37 </pre>
38 
39 The addon should sit in `openFrameworks/addons/ofxKinect/`.
40 
41 #### Which version to use?
42 
43 If you are using a stable version (0062, 007, ...) of OpenFrameworks then you want to use a git tag of ofxKinect for that version. You can select the tag in the Github menu or clone and check it out using git.
44 
45 For example, the following commands will clone ofxKinect and switch to the OF 0062 tagged version:
46 <pre>
47 git clone git://github.com/ofTheo/ofxKinect.git
48 cd ofxKinect
49 git checkout 0062
50 </pre>
51 
52 #### Using the latest ofxKinect
53 
54 The master branch of ofxKinect will work with the current master of OpenFrameworks and can be considered unstable. The develop branch is used for testing new features and bugfixes before they are applied to master.
55 
56 If you want to work with the latest unstable (still in development) ofxKinect, download the source from the develop branch [https://github.com/ofTheo/ofxKinect/tree/develop](https://github.com/ofTheo/ofxKinect/tree/develop) or via git clone:
57 <pre>
58 git clone git://github.com/ofTheo/ofxKinect.git -b develop
59 </pre>
60 
61 Warning: The develop branch will be in flux, so don't be surprised if things do not always work as expected!

63 Running the Example Project
64 ---------------------------
65 
66 An exmaple project is provided in the `kinectExample` folder. As of OF 0.8.0, ofxKinect no longer ships with the platform poject files. Simply use the OpenFrameworks ProjectGenerator in `apps/projectGenerator` to generate the Xcode, VS2012, CodeBlocks projects and/or Makefiles by pointing it to the `kinectExample` folder and making sure to include the following addons:
67 
68 * ofxKinect (duh)
69 * ofxOpenCv (for blob tracking in the example, not required by ofxKinect itself.)
70 
71 ### OSX
72 
73 Xcode4: Open the Xcode project, select the "kinectExample Debug" scheme, and hit "Run".
74 
75 ### Linux
76 
77 Install the libusb-1.0 library. On Ubuntu, you can do this with:
78 <pre>
79 sudo apt-get install libusb-1.0-0-dev
80 </pre>
81 
82 Open the Code::Blocks .cbp and hit F9 to build. Optionally, you can build the example with the Makefile.
83 
84 To run it, use the terminal:
85 <pre>
86 make
87 cd bin
88 ./example_debug
89 </pre>
90 
91 Also, you can add a set of udev rules which allow you to run a kinect app without root privileges:
92 
93  sudo libs\libfreenect\platform\linux\udev\51-kinect.rules /etc/udev/rules.d
94 
95 ### Windows
96 
97 Precompiled libfreenect Kinect drivers and libusb-win32 libs are included for Windows.
98 
99 Make sure to install or update the libfreenect Kinect camera, motor, and audio drivers through Windows Device Manager by pointing it to the driver folder:
100 <pre>
101 libs/libfreenect/platform/windows/inf
102 </pre>
103 
104 You may need to manually update each driver individually if you've plugged it in before. ofxKinect will not work if the drivers are not installed.
105 
106 **NOTE**: You cannot use the OpenNI drivers and the libfreenect drivers included with ofxKinect at the same time. You must manually uninstall one and reinstall the other in the Device Manager. Sorry, that's just how it is. :P

108 How to Create a New ofxKinect Project
109 -------------------------------------
110 
111 To develop your own project based on ofxKinect, simply copy the example project and rename it. You probably want to put it in your apps folder, for example, after copying:
112 
113 `openFrameworks/addons/ofxKinect/kinectExample/ => openFrameworks/apps/myApps/example/`
114 
115 Then after renaming:
116 
117 `openFrameworks/apps/myApps/myKinectProject/`
118 
119 ### Mac (Xcode):
120 
121 Rename the project in Xcode (do not rename the .xcodeproj file in Finder!): Slow double click the project name and rename (Xcode4)
122 
123 ### Codeblocks (Win & Linux):
124 
125 Close Codeblocks and rename the *.cbp and *.workspace files to the same name as the project folder. Open the workspace and readd the renamed project file.
126 
127 ### Windows (Visual Studio):
128 
129 Close Visual Studio and rename the *.sln file. Open the solution, right click on the project in the project broswer and select "Rename ...".

131 Adding ofxKinect to an Existing Project
132 ---------------------------------------
133 
134 If you want to add ofxKinect to another project, you need to make sure you include the src folder and the :
135 <pre>
136 openFrameworks/addons/ofxKinect/src
137 </pre>
138 as well as search paths for the addons (ofxOpenCv) and libs (freenect, libusb) required by ofxKinect.
139 
140 Don't forget to include the ofxOpenCv addon sources and oepncv libraries.
141 
142 ### Mac (Xcode):
143 
144 In the Xcode project browser:
145 
146 * add the ofxKinect sources to the project:
147  * create a new group "ofxKinect"
148  * drag these directories from ofxKinect into this new group: ofxKinect/src & ofxKinect/libs
149 * add a search path to the libusb headers: `../../../addons/ofxKinect/libs/libusb/include/libusb-1.0` to your project Target build settings
150  * Xcode4: add the lib path to your Project.xconfig, see the example
151 * add the path to the libusb precompiled library: `../../../addons/ofxKinect/libs/libusb-1.0/lib/osx/usb-1.0.a`
152  * Xcode4: add the lib path to your Project.xconfig, see the example
153 
154 ### Linux (Makefiles & Codeblocks):
155 
156 Edit the Makefile config files:
157 
158 * edit addons.make in your project folder and add the following line to the end of the file:
159  <pre>ofxKinect</pre>
160 * edit config.make in your project folder and change the lines for USER_LIBS to:
161  <pre>
162  USER_LIBS = -lusb-1.0
163  </pre>

217 Notes

219 
220 ### Using multiple kinects
221 
222 ofxKinect supports multiple kinects, however stability is based on the bandwidth of your usb hardware. If you only require the depth image, it is recommended to disable the video grabbing of the rgb/ir images from your devices:
223 <pre>
224 kinect.init(false, false); // disable video image (faster fps)
225 </pre>
226 
227 For Kinect4Windows, Microsoft states that only 2 Kinects can be supported on the same USB bus. In practice on OSX, this proves to be the case as, even with the RGB images disabled, there are transfer errors using ofxKinect and 3 Kinects simultaneously. If you need to support many Kinects, you will probably need to add extra USB controllers to your machine â€¦
