25 /*
26  * A TestResult collects the results of executing a test case. It is an
27  * instance of the Collecting Parameter pattern.
28  *
29  * The test framework distinguishes between failures and errors.
30  * A failure is anticipated and checked for with assertions. Errors are
31  * unanticipated problems signified by exceptions that are not generated
32  * by the framework.
33  *
34  * TestResult supplies a template method 'setSynchronizationObject ()'
35  * so that subclasses can provide mutual exclusion in the face of multiple
36  * threads. This can be useful when tests execute in one thread and
37  * they fill a subclass of TestResult which effects change in another
38  * thread. To have mutual exclusion, override setSynchronizationObject ()
39  * and make sure that you create an instance of ExclusiveZone at the
40  * beginning of each method.
41  *
42  * see Test
43  */
